## 1. 系统总览

Modern Ops 的代码架构分为 **14 个主要子系统**，全部通过抽象基类构建，具备高度的可扩展性。

### 核心设计原则

| 原则 | 实现方式 |
|------|----------|
| **单一职责** | 每个基类只管理一类职责（CCItem管物品信息，BaseWeapon管武器数值） |
| **组合优于继承** | 战斗控制器通过接口组合（IBattlePlayer 依赖注入） |
| **数据驱动** | 所有数值来自 JSON 配置（CRCD 系列），运行时对象只持有引用 |
| **网络同步** | Photon PUN + 自定义 DataContract 层，所有战斗事件走 Photon 事件码 |
| **对象池** | WeaponsEffectsPool 命名空间统一管理 VFX/Projectile 生命周期 |

## 2. 物品系统

物品系统是全游戏内容（武器、皮肤、KillStreak、能力等）的统一管理基础。

### 2.1 CCItem — 通用物品基类

```
文件：Assets/Scripts/Assembly-CSharp/CCItem.cs
命名空间：（全局）
```

**作用：** 所有可购买/装备的游戏内物品的通用基类。武器、皮肤、KillStreak、手雷、能力均继承自此类。

```csharp
public class CCItem : INewlyAddedCCItem
{
    // ── 标识 ──────────────────────────────────────────
    public int    ItemID       { get; protected set; }  // 唯一数值ID（对应JSON的 i 字段）
    public string SystemName   { get; protected set; }  // 系统内部名（用于资源加载路径）
    public string Name         { get; protected set; }  // 本地化显示名
    public string Desc         { get; protected set; }  // 本地化描述
    public string SubTypeDesc  { get; protected set; }  // 子类型描述（武器类型文字）

    // ── 图标 ──────────────────────────────────────────
    public string IcoFileName    { get; }               // 小图标资源名
    public string IcoBigFileName { get; }               // 大图标资源名

    // ── 商店 ──────────────────────────────────────────
    public ShopCost  ShopCost      { get; protected set; }  // 当前价格（金币/钻石）
    public ShopCost  PrevShopCost  { get; protected set; }  // 折扣前原价
    public short     Discount      { get; }                 // 折扣百分比（0-100）
    public BankSetPack BankSetRel  { get; set; }            // 关联礼包

    // ── 解锁条件 ──────────────────────────────────────
    public uint NeedLvl       { get; protected set; }   // 需要玩家等级
    public bool IsLocked      { get; }                  // 是否锁定
    public bool IsBuyed       { get; }                  // 是否已购买
    public bool IsEnabled     { get; }                  // 是否在游戏中启用

    // ── 稀有度 ──────────────────────────────────────
    public ItemRarityType RarityType { get; set; }
    // Common / Uncommon / Rare / Epic / Legendary

    // ── 状态 ──────────────────────────────────────────
    public ItemStateType State { get; protected set; }
    // Normal / New / Updated / NewlyUnlocked

    // ── 蓝图 ──────────────────────────────────────────
    public Blueprint BlueprintRel { get; set; }         // 关联蓝图

    // ── 附加值 ──────────────────────────────────────────
    public readonly AdditionalValues AdditionalValues;   // 扩展属性字典

    // ── 虚方法 ──────────────────────────────────────────
    public virtual CCItemType ItemType => default;       // 枚举类型标识
    public virtual bool IsUsedByLocalUser();             // 是否被本地用户装备
    public virtual void EquipToLocalUser(bool delayedRequest = false);  // 装备到本地用户
    public virtual bool CanBeUsed();                     // 是否可被使用
    protected virtual void SetLocalization();            // 设置本地化文本（子类重写）
}
```

### 2.3 BaseUpgrade — 升级链节点

```
文件：Assets/Scripts/Assembly-CSharp/BaseUpgrade.cs
```

**作用：** 武器/KillStreak 升级系统的抽象节点。每个升级等级对应一个 `BaseUpgrade` 实例，通过 `PrevLevel`/`NextLevel` 双向链表串联。

```csharp
public abstract class BaseUpgrade : IItemWithDuration
{
    // ── 货币类型枚举 ─────────────────────────────────
    public enum UpgradeCurrency
    {
        None              = 0,   // 无需货币（特殊获取）
        OnlyMoney         = 1,   // 仅需金币
        BlueprintsAndMoney = 2,  // 蓝图 + 金币
    }

    // ── 静态字段（只读，来自JSON） ────────────────────
    public readonly int           UpgradeID;
    public readonly int           NeedLvl;                   // 需要玩家等级
    public readonly int           BlueprintID;
    public readonly int           BlueprintCountToUpgrade;
    public readonly int           DelaySecNeededToUpgrade;   // 升级耗时（秒）
    public readonly ShopCost      Cost;                      // 升级花费
    public readonly UpgradeCurrency Currency;

    // ── 链表关系 ──────────────────────────────────────
    public BaseUpgrade PrevLevel { get; }      // 前一等级（null=最低级）
    public BaseUpgrade NextLevel { get; }      // 后一等级（null=最高级）
    public int         Rank      { get; }      // 当前等级排名（0-based）

    // ── 物品引用（升级前/后） ─────────────────────────
    public int     SourceShopItemID  { get; protected set; }
    public int     RewardShopItemID  { get; protected set; }
    public CCItem  SourceShopItem    { get; }               // 升级前的物品
    public CCItem  RewardShopItem    { get; }               // 升级后的物品

    // ── 运行时状态 ───────────────────────────────────
    public bool IsEnabled;
    public bool IsLocked;
    public bool IsNewlyAvailable;
    public bool IsInProgress;           // 升级计时器是否运行中
    public bool IsDelayedUpgrade;       // 是否使用延迟升级模式
    public bool IsDelayPassed;          // 延迟是否已过（可立即完成）
    public ShopCost ForceFinishUpgradeCost;  // 立即完成的加速花费

    // ── 抽象方法 ──────────────────────────────────────
    public abstract bool         IsAnyLevelAlreadyBought();
    public abstract bool         IsAnyUpLevelAlreadyBought();
    public abstract BaseUpgrade  GetLastBuyedLevel();        // 获取最后已购买等级
    public abstract BaseUpgrade  GetLastLevel();             // 获取最高等级节点
    public abstract int          GetLastRankIndex(bool onlyEnabled = true);
}
```

**升级链结构示意：**

```
BaseUpgrade[Lv.1] ←PrevLevel/NextLevel→ BaseUpgrade[Lv.2] → ... → BaseUpgrade[Lv.N]
      ↓                                        ↓
SourceShopItem(原始物品)              SourceShopItem(Lv.1物品)
RewardShopItem(Lv.1物品)             RewardShopItem(Lv.2物品)
```

### 2.4 BaseSlot\<T\> — 装备槽容器

```
文件：Assets/Scripts/Assembly-CSharp/BaseSlot.cs
```

**作用：** 泛型装备槽管理器，持有 N 个槽位，每个槽位存一个 CCItem 子类实例。

```csharp
public class BaseSlot<T> where T : CCItem
{
    protected class SlotData
    {
        public int ItemID;      // 物品ID（-1=空槽）
        public T   ItemModel;   // 物品数据对象
    }

    protected Dictionary<int, SlotData> _slotsData;  // key=槽号

    public int SlotsCount { get; }                   // 槽位总数
    public T   this[int slot] { get; }               // 索引器：获取指定槽物品

    protected void InitSlotsDict(int slotsCount);    // 初始化槽字典
    protected virtual void SyncWithInventory(List<T> inventoryListOfItems);  // 与背包同步
    protected bool Set(int slot, T itemModel, ref bool haveChanges);         // 设置槽物品
    public T   GetSlotItemModel(int slot);           // 获取槽物品（同索引器）
    public virtual int GetSlotID(int slot);          // 获取槽中物品ID
}
```

## 附录 C：迁移指南（关键注意事项）

### 1. 网络层依赖

本项目深度绑定 **Photon PUN**（`ExitGames.Client.Photon`）。迁移时需关注：
- 所有战斗事件（KillStreak/击杀/得分）都通过 Photon Event Code 广播
- `Hashtable` 序列化（Photon特有），需替换为自定义序列化方案
- `ActorID` 是玩家唯一标识（房间内），迁移时对应自定义网络层的 PlayerID

### 2. 方法体缺失

AssetRipper 导出的所有方法体均返回默认值（null/false/0）。以下类的完整实现需要根据签名和上下文重新实现：
- 所有 `abstract` 方法（标注清晰，已知签名）
- `BaseLoadManager.Load()` / `AgainLoad()`（参考CRC缓存机制文档）
- `BaseGameMode`（92个方法，大量业务逻辑）

### 3. 第三方框架依赖

| 框架 | 用途 | 可替换方案 |
|------|------|----------|
| NodeCanvas | AI行为树 | BehaviorDesigner / 自制行为树 |
| I2 Localization | 多语言 | Unity Localization Package |
| Unity Playables | 自定义动画层 | 直接使用 Animator / 自制 |
| Photon PUN | 多人网络 | Mirror / Netcode for GameObjects |

### 4. 数据驱动架构

所有游戏内容数值存储在 `CRCD{N}.json` 配置文件中，运行时通过 CRC 校验机制从服务器热更新。迁移时建议：
- 保留数据驱动思路，将数值与代码分离
- `CRCD13.json`（KillStreak）、`CRCD27.json`（武器）是最核心的配置

### 5. CCItem 是一切的起点

所有游戏内容对象都继承 `CCItem`。迁移时建议先实现 CCItem 基类及其字段系统，再逐步添加各类子系统（武器/KS/手雷等）。
